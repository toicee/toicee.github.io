iOS Safari에서 entity code로 dash 표현시

created
2019년 3월 11일
updated
2019년 3월 15일
category
worklog
tags
entitymarkup 

&#8209; &minus; &dash; &mdash; &ndash;

할인금액을 써야되는 경우 ‘-‘의 표현을 &minus;로 사용했었는데 디자인보다 길다하여 &dash;로 바꾸고 맥에서 확인해보고 별 문제 없길래 대수롭지 않게 넘어갔는데 iOS에서 확인해보니 언더스코어처럼 되어 있었다. 화들짝 놀라 &#8209;로 급 변경.

참고 : https://www.w3schools.com/charsets/ref_utf_punctuation.asp

추가로 자주 사용하는 엔티티코드 변환 사이트

https://www.evotech.net/articles/testjsentities.html

주로 CSS에 content값에 넣을 값들을 찾을 때 자주 쓰는데 여기가 좋은 점은 내가 원하는 캐릭터를 제일 밑에칸에 써넣으면 마크업, CSS, JS용으로 한번에 컨버팅 해준다는 점이다.

https://github.com/mathiasbynens/he

터미널에서 결과값을 받을 수 있기도 한데, 엔티티코드를 넣을 때는 주로 터미널 작업이랑 연계되어지는 작업이 대다수 아니기 때문에 웹사이트를 더 자주 쓰게 됨.
